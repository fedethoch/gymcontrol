# Definir lineamientos de instrucciones optimizadas en tokens: mantener skills y agentes breves y utiles

**Grupo:** G3 - Skills y agentes  
**Orden:** 6  
**Esfuerzo:** Bajo  
**Modelo recomendado:** gpt 5.4 mini

## Objetivo

Definir reglas concretas para que las skills y agentes del proyecto usen instrucciones cortas, claras y suficientes, sin texto innecesario que incremente el costo en tokens.

## Pasos

1. Establecer principios de brevedad y foco para skills y agentes.
2. Definir que informacion debe incluirse siempre y que informacion debe evitarse.
3. Definir una longitud o nivel de detalle razonable para estas instrucciones.
4. Registrar lineamientos accionables y simples de revisar.

## Criterios de aceptacion

- Existen reglas claras para optimizar tokens.
- Las reglas son faciles de aplicar al redactar skills o agentes.
- Se evita texto redundante o decorativo sin perder utilidad.

## Resolucion

Se definieron lineamientos concretos para redactar skills breves, enfocadas y con bajo costo de tokens, priorizando referencias al contexto ya documentado del repo.

## Validacion

- Verificar que la optimizacion no sacrifique claridad operativa.
