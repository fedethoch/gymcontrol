# Identificar tareas recurrentes que justifican skills o agentes: separar necesidad real de conveniencia

**Grupo:** G3 - Skills y agentes  
**Orden:** 1  
**Esfuerzo:** Bajo  
**Modelo recomendado:** gpt 5.4 mini

## Objetivo

Identificar que tareas del proyecto se repiten lo suficiente o tienen suficiente complejidad como para justificar la creacion de una skill o de un agente dedicado.

## Estado

## Pasos

1. Leer el alcance del MVP y el plan general.
2. Listar tareas recurrentes de definicion, implementacion, revision o validacion.
3. Detectar cuales realmente se benefician de una instruccion reusable o de una separacion en agente.
4. Dejar una lista preliminar de candidatos justificados.

## Criterios de aceptacion

- Existe una lista de tareas candidatas a skill o agente.
- Cada candidata tiene una razon concreta para existir.
- No se confunden tareas ocasionales con tareas recurrentes.

## Resolucion

Se identificaron como candidatas reales las tareas de lectura de contexto del repo, ejecucion de specs, uso correcto del MCP de Supabase y validacion final de cambios.

## Validacion

- Verificar que las tareas identificadas aparezcan varias veces a lo largo del plan.
- Verificar que la lista no se infle con ideas accesorias.
