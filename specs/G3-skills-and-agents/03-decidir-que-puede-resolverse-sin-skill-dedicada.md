# Decidir que puede resolverse sin skill dedicada: evitar fragmentacion innecesaria

**Grupo:** G3 - Skills y agentes  
**Orden:** 3  
**Esfuerzo:** Bajo  
**Modelo recomendado:** gpt 5.4 mini

## Objetivo

Determinar que partes del trabajo del MVP pueden resolverse directamente con contexto del repo y buenos specs, sin necesidad de crear una skill o un agente aparte.

## Estado

Completado.

## Archivos

- resultados de los specs `01` y `02` de este grupo
- `PLAN.md`
- estructura actual de `specs/`

## Pasos

1. Revisar las tareas que quedaron fuera de la lista refinada.
2. Identificar como se resolveran usando documentacion, specs y flujo normal de trabajo.
3. Registrar explicitamente que no requieren una skill o agente propio.
4. Dejar criterio para futuras reevaluaciones.

## Criterios de aceptacion

- Queda claro que tareas no ameritan skill dedicada.
- Se reduce el riesgo de crear demasiadas piezas auxiliares.
- El enfoque mantiene el trabajo centralizado y legible.

## Resolucion

Se definio que gran parte del MVP puede resolverse sin skill dedicada usando los specs, la documentacion base y el flujo normal del repo. Solo se mantienen como candidatas las piezas realmente reutilizables.

## Validacion

- Verificar que la mayoria del trabajo del MVP siga siendo resoluble con specs y documentos base.
- Verificar que no se este rechazando una skill realmente necesaria.

Completado.
