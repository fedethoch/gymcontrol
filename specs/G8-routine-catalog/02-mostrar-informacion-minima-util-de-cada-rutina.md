# Mostrar informacion minima util de cada rutina: facilitar la eleccion desde el catalogo

**Grupo:** G8 - Catalogo de rutinas  
**Orden:** 2  
**Esfuerzo:** Bajo  
**Modelo recomendado:** gpt 5.4 mini

## Objetivo

Definir y mostrar la informacion minima necesaria para que el usuario entienda y compare las rutinas del catalogo.

## Estado

Pendiente.

## Archivos

- vista catalogo
- datos de rutinas
- `docs/ARCHITECTURE.md`
- `PLAN.md`

## Pasos

1. Definir que datos minimos debe mostrar cada rutina en el listado.
2. Incorporar esa informacion en la UI del catalogo.
3. Evitar sobrecargar la tarjeta o fila de rutina con datos secundarios.
4. Mantener consistencia visual y claridad de lectura.

## Criterios de aceptacion

- Cada rutina muestra informacion suficiente para ser identificada.
- El usuario puede comparar opciones sin entrar obligatoriamente al detalle.
- La informacion mostrada es simple y relevante.

## Resolucion

Completar al ejecutar este paso.

## Validacion

- Verificar que la vista no quede vacia de contexto ni sobrecargada.
- Verificar que la informacion mostrada sea coherente con el modelo real de rutina.
