# Specs

Esta carpeta contiene la estructura operativa de trabajo del proyecto en formato de specs ejecutables por paso.

## Objetivo

Permitir trabajar el MVP de GymControl de forma ordenada, incremental y con contexto acotado.

Cada grupo representa una etapa del proyecto. Dentro de cada grupo, cada archivo `.md` representa un paso individual que puede ejecutarse de manera aislada.

## Regla de uso

La forma esperada de trabajo es:

1. leer este archivo
2. leer `README.md` del grupo correspondiente
3. leer el spec puntual a ejecutar
4. ejecutar el spec
5. actualizar documentos, codigo o estado segun corresponda

## Convencion de avance

Cuando se termine un spec, siempre se debe indicar cual sigue segun el orden vigente.

La indicacion del siguiente paso debe incluir tambien el `Modelo recomendado` y el `Esfuerzo` de ese spec. Esto mantiene el contexto operativo completo en un solo mensaje.
